# 🚀 Automated File Transfer between S3 Buckets using AWS Lambda

The DevOps team is working on automating file management between two S3 buckets. The task is to create a public S3 bucket for file uploads and a private S3 bucket for securely storing the files. A Lambda function will be triggered automatically whenever a file is uploaded to the public S3 bucket, which will copy the file to the private bucket. Additionally, logs of the operation will be stored in a DynamoDB table.

The logs should include details such as:
- Source bucket
- Destination bucket
- Object key of the copied file

This setup improves both **security** and **visibility** of file transfers within the infrastructure.

## 📌 Tasks

- Create a public S3 bucket named **xfusion-public-25474** and allow public access to its objects.
- Create a private S3 bucket named **xfusion-private-12290** and ensure public access is blocked.
- Create a Lambda function named **xfusion-copyfunction** which is triggered when a file is uploaded to the public bucket.
- The Lambda function will copy the uploaded file to the private bucket.
- Create an IAM role named **lambda_execution_role** and attach the required policies.
- Link this role to the Lambda function.

## 🧠 Lambda Configuration

A file named **lambda-function.py** already exists in the `/root/` directory on the AWS client host.

Before deploying the Lambda function, update the following placeholders in the script:

- `REPLACE-WITH-YOUR-DYNAMODB-TABLE`
- `REPLACE-WITH-YOUR-PRIVATE-BUCKET`

Replace them with:

- **xfusion-S3CopyLogs**
- **xfusion-private-12290**

## 🗄️ DynamoDB

Create a DynamoDB table with the following configuration:

- **Table Name:** xfusion-S3CopyLogs  
- **Partition Key:** LogID (String)

This table will store logs generated by the Lambda function, including:

- Source bucket name
- Destination bucket name
- Object key

## 🧪 Testing the Workflow

To test the automation:

1. Upload the file **sample.zip** located in `/root` on the client host to the public bucket **xfusion-public-25474**.
2. The Lambda function should automatically trigger.
3. The uploaded file should be copied to the private bucket **xfusion-private-12290**.

## ✅ Verification

Verify the setup by performing the following checks:

- Confirm that **sample.zip** exists in the private S3 bucket.
- Check the **xfusion-S3CopyLogs** DynamoDB table to ensure that a log entry was created containing:
  - Source bucket name
  - Destination bucket name
  - Object key
